In order for reactive data refreshing, mutators and many validators to work, you will have to 'link' form elements together. In essence this simply means declaring which questions rely on each other.

In the above example we identify the birdOfTheYear element as a linked element to the dynamic toggle. This means that upon value or status changes of the toggle, it will execute each validator and mutator function in sequence and pass the linked elements as argument. If the refreshOnValueChange property is set to true, it will also execute the refresh method of the linked element's data source (only available for DynamicSelect elements that have an AbstractObservableDataSource set as data source.)
